Стварэнне новай справаздачы

Ёсць 2 варыянты.

Стварэнне справаздачы з нуля

  1. Выкарыстоўваючы дрэва навігацыі, пстрыкніце правай кнопкай мышы па надпісе Справаздачы.
  2. Абярыце пункт меню: Новая пустая справаздача.
  3. Увядзіце назву новай групы справаздачаў і новую назву.
  4. За неабходнасцю, перайменуйце справаздачу, пстрыкнуўшы правай кнопкай мышы на пункце меню: Перайменаваць справаздачу.
  5. Будзе адлюстравана пяць ўкладак: Вынік, SQL, Lua, Шаблон і Апісанне.
  6. Абярыце ўкладку: SQL, устаўце скрыпт SQL у акно рыдактара, пераканайцеся, што скрыпт SQL заканчваецца кропкай з коскай, а затым націсніце кнопку Захаваць.
  7. Націсніце кнопку Тэст, каб выканаць скрыпт SQL.
  8. Калі тэст пройдзены, кнопка Стварыць шаблон стане даступнай. Націсніце яе, каб стварыць HTML-шаблон для справаздачы.
  9. Націсніце кнопку Выканаць, каб атрымаць HTML-справаздачу ў акне Вынік.

Параметры справаздачы

Для MMEX версіі 1.4.0 ёсць магчымасць пераноса дат у сцэнарый SQL справаздачы. Даступныя чатыры варыянты: &single_date, &begin_date, &end_date, &budget_years. У выпадку, калі SQL змяшчае адзін з параметраў вышэй, ён будзе заменены датай з календара. Напрыклад, гэты сцэнар SQL вяртае ўсе рэшткі рахункаў за паказаную дату: 

with b as  (
    select ACCOUNTID, STATUS
        , (case when TRANSCODE = 'Deposit' then TRANSAMOUNT else -TRANSAMOUNT end) as TRANSAMOUNT
        , TRANSDATE
    from CHECKINGACCOUNT
    union all
    select TOACCOUNTID, STATUS, TOTRANSAMOUNT ,TRANSDATE
    from CHECKINGACCOUNT
    where TRANSCODE = 'Transfer'
    )
select a.ACCOUNTNAME,
total(TRANSAMOUNT) + a.INITIALBAL as Balance
from ACCOUNTLIST as a, b
where a.STATUS = 'Open'
    and b.ACCOUNTID = a.ACCOUNTID
    and b.STATUS <> 'V'
    and b.TRANSDATE <= '&single_date'
group by a.ACCOUNTNAME order by a.ACCOUNTNAME asc;
